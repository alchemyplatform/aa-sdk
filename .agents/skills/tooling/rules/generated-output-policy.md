# Generated Output Policy

Do not hand-edit generated outputs as source.

## Why

Build and docs generation write committed output that CI checks for freshness.
Manual edits drift from the generators and get overwritten by `pnpm run build`
or `pnpm run docs:sdk`.

## Generated or Build Output

- `dist/**`: package build output, ignored by git.
- `node_modules/**`, `.turbo/**`, coverage output: local build/cache output.
- `docs/pages/reference/**`: committed TypeDoc output generated by
  `pnpm run docs:sdk`.

## Good

For reference docs, edit the package source, TSDoc, `typedoc.json`,
`custom-frontmatter.mjs`, or `scripts/generate-typedoc-yaml.ts`, then run:

```bash
pnpm run docs:sdk
pnpm run lint:write
git diff -- docs/pages/reference/ docs/docs.yml
```

## Bad

Editing `docs/pages/reference/**/*.mdx` to fix API docs without changing the
source that TypeDoc reads.

## Exceptions

Small generated-output diffs are expected after source changes. Commit them
when CI would otherwise report reference docs drift.
