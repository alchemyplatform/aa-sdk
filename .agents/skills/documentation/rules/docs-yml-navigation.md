# docs.yml Navigation

Keep reference navigation generated by `scripts/generate-typedoc-yaml.ts`.

## Why

The script reads `docs/pages/reference`, uses `PACKAGES_INCLUDED_IN_NAV`, and
updates the SDK Reference section in `docs/docs.yml`. Main-branch hooks trigger
the external docs indexer when `docs/docs.yml` changes.

## Good

- Update `PACKAGES_INCLUDED_IN_NAV` when intentionally adding or removing a
  package from reference navigation.
- Run `pnpm run docs:sdk` after TypeDoc entry point or public export changes.
- Review both `docs/pages/reference/**` and `docs/docs.yml` diffs together.

## Bad

Manually editing generated SDK Reference entries in `docs/docs.yml` while
leaving the generator unchanged.

## Exceptions

Hand-authored navigation outside the generated SDK Reference section can be
edited directly when it is not controlled by the generator.
